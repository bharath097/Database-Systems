# Database Systems – End-to-End Learning & Implementation

# Overview

In this repository, I documented my hands-on work across multiple database systems assignments where I implemented concepts ranging from SQL querying to advanced database internals.

Through this work, I didn’t just learn how to write queries — I understood how databases actually function behind the scenes, how data is stored, accessed, secured, and optimized for real-world applications.


# What I Built & Learned (Through Implementation)

I worked across multiple layers of database systems:

 designed and queried relational databases using SQL, combining multiple tables and extracting meaningful insights from structured data.

 connected databases with Java using JDBC and built programs that execute queries dynamically, simulating how backend systems interact with databases.
 
 implemented hierarchical data processing using recursion, both through SQL (`WITH RECURSIVE`) and database functions (PL/pgSQL), which helped me understand how complex relationships are handled in production systems.
 
 analyzed how databases manage access control using GRANT and REVOKE, including how privileges propagate and how revocation affects dependent users.
 
 studied transaction isolation and concurrency issues such as dirty reads, phantom reads, and lost updates, and understood how systems ensure consistency under multiple simultaneous operations.
 
 explored how data is physically stored using fixed-length fields, pointers, and record structures, and how design decisions impact performance and scalability.
 
 implemented recursive database functions to traverse and compute values over hierarchical structures, similar to tree/graph processing inside the database.
 
 worked on indexing and hashing techniques including B+ trees, extensible hashing, and linear hashing, understanding how databases optimize search and storage efficiency at scale.
 
 calculated storage requirements for dense and sparse indexes, analyzing trade-offs between performance and memory usage.


# How I Approach Problem Solving

For every problem, I followed a structured engineering approach:

I first understood the data model and relationships between entities.
I broke down the problem into smaller logical steps.
I implemented solutions incrementally, validating outputs at each stage.
I handled edge cases such as missing data, nested relationships, and concurrency conflicts.
I optimized solutions by considering performance, storage, and scalability.
